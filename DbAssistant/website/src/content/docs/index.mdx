## What is DbAssistant?

A single, modular tool that manages SQL and NoSQL databases across AWS,
Azure, GCP, and on-prem — with a desktop UI, a terminal UI, a web UI, a
scriptable CLI, and a fully featured REST API that all share the same
business logic.

<CardGrid stagger>
  <Card title="Universal database support" icon="seti:db">
    Oracle, MySQL, MariaDB, PostgreSQL, SQL Server, MongoDB, AWS DocumentDB, SQLite — uniform CRUD, schema, and object browsing.
  </Card>
  <Card title="Five surfaces, one logic core" icon="rocket">
    Whatever you can do in the desktop UI, you can do in the TUI, web UI, CLI, or REST API — tested end-to-end for parity.
  </Card>
  <Card title="AI SQL with RAG + a local model" icon="open-book">

  <Card title="AWS · Azure · GCP monitoring" icon="cloud-download">

  <Card title="Ship a single module" icon="puzzle">

## Quick install

Requires Python 3.10+. Online bundles are ~500 KB; offline bundles
(~1 GB) include wheels for macOS, Linux, and Windows across Python
3.10, 3.11, and 3.12.

<Tabs>
<TabItem label="macOS / Linux">

```bash
unzip dbassistant-full-1.0.0.zip
cd dbassistant-full-1.0.0
./install.sh
./run.sh
```

</TabItem>
<TabItem label="Windows">

```batch
:: extract dbassistant-full-1.0.0.zip
cd dbassistant-full-1.0.0
install.bat
run.bat
```

</TabItem>
<TabItem label="Offline (no internet)">

```bash
unzip dbassistant-full-1.0.0-offline.zip
cd dbassistant-full-1.0.0-offline
./install.sh --offline
```

</TabItem>
</Tabs>

## Try the CLI in 30 seconds

```bash
python dbtool.py modules
python dbtool.py connections add --name prod --type PostgreSQL \
    --host db.example.com --user app --port 5432 --db appdb
python dbtool.py query --conn prod --sql "SELECT version()"
python dbtool.py ai --conn prod "show the top 5 customers by revenue"
python dbtool.py monitor --conn prod --interval 30
```

## Or use the REST API

```bash
python dbtool.py api --host 0.0.0.0 --port 8000

curl http://localhost:8000/api/health
curl -X POST http://localhost:8000/api/query \
    -H "Content-Type: application/json" \
    -H "X-API-Key: $DBTOOL_API_KEY" \
    -d '{"connection":"prod","sql":"SELECT 1"}'
curl http://localhost:8000/api/metrics/prod
```

## Designed for production

<CardGrid stagger>
  <Card title="Hermetic file layout" icon="seti:folder">
    Everything lives under <code>~/.dbassistant/</code> with subfolders for keys, connections, runtime, and session data. <code>DBASSISTANT_HOME</code> overrides everything for tests and containers.
  </Card>
  <Card title="Idempotent migration" icon="seti:json">
    Copy-then-rename migration with file locking and a version stamp. Safe across upgrades; never leaves partial state on disk.
  </Card>
  <Card title="Tested end-to-end" icon="approve-check">
    Full pytest suite covering UI, CLI, and REST API. Integration tests with real databases and cloud profiles.
  </Card>
  <Card title="Cross-platform shipper" icon="seti:zip">
    <code>shipper.sh</code> builds lean (~500 KB) or offline (all platforms, all Python 3.10/3.11/3.12 wheels) bundles. Per-module bundles supported.
  </Card>
</CardGrid>
