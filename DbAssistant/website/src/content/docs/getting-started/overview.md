DbAssistant is a production-grade database management tool with five
interchangeable interfaces sharing one logic core.

## Five surfaces, one logic core

| Surface | When to use it | Entry point |
|---------|----------------|-------------|
| **Desktop UI** (Tkinter) | Interactive work — browsing schema, running queries, monitoring | `conDbUi.py` / `python dbtool.py ui` |
| **Terminal UI** (Textual) | Remote/SSH sessions, keyboard-driven work | `python dbtool.py tui` |
| **Web UI** (FastAPI + SPA) | Shared/team access from a browser | `python dbtool.py webui` → `/ui/` |
| **CLI** | Scripting, CI, automation | `python dbtool.py <command>` |
| **REST API** | Integration with other services, dashboards, custom front-ends | `python dbtool.py api` → `/docs` |

All five surfaces dispatch to the same service layers
(`common/headless/db_service.py` and `app/headless/db_service.py`), so feature
parity is guaranteed across the Desktop UI, TUI, Web UI, CLI, and API.

## Modular architecture

The tool ships as a **shared core** plus **four optional modules**:

| Module | Folder | Purpose |
|--------|--------|---------|
| **Data Migration** | `schema_converter/` | Schema convert, data transfer, validation |
| **AI Query Assistant** | `ai_query/` | Natural-language → SQL, RAG retrieval, local LLM |
| **Monitoring** | `monitoring/` | Local + OS + cloud monitoring with thresholds and alerts |
| **App Builder** | `ai_assistant/app_builder/` | Generate apps from scratch, a codebase, or a database |

The core (connections, SQL editor, object browser, engine registry) is
always present. Each module plugs in through a `manifest.py` declaring
its CLI commands, API router, and optional UI tab. You can ship the
full tool or any single module independently — see
[Modules & shipping](/architecture/modules/).

## Supported databases

| Engine | Driver |
|--------|--------|
| Oracle | `oracledb` |
| MySQL | `mysql-connector-python` |
| MariaDB | `mysql-connector-python` |
| PostgreSQL | `psycopg2-binary` |
| SQL Server / Azure SQL | `pymssql` |
| MongoDB | `pymongo` |
| AWS DocumentDB | `pymongo` (TLS required) |
| SQLite | built-in |

Each engine exposes **capability metadata** (query language, schema
conversion support, transactions). The UI, CLI, and API use this to
enable or disable features automatically.

## Supported clouds

| Cloud | Service | API |
|-------|---------|-----|
| AWS | RDS, Aurora | CloudWatch + Performance Insights |
| Azure | SQL, MySQL, PostgreSQL, MariaDB, Cosmos, Redis | Azure Monitor |
| GCP | Cloud SQL | Cloud Monitoring (System Insights) |

## Where everything lives

Everything generated by the tool lives under `~/.dbassistant/`:

```
~/.dbassistant/
├── keys/             # encryption keys (db, cloud, monitor)
├── connections/      # encrypted profiles (db, cloud, monitor)
├── runtime/          # daemon PID, metrics.json, logs
├── session/          # AI sessions, dashboard layout
└── version           # layout migration stamp
```

The base directory can be overridden by setting `DBASSISTANT_HOME`,
which is essential for tests and isolated installs.
