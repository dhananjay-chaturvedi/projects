# Deployment

The site builds to fully static HTML/CSS/JS тАФ drop `dist/` on any
static host.

## Option 1 тАФ GitHub Pages (automatic)

A workflow at `.github/workflows/website.yml` builds on every push to
`main` that touches `website/**` and deploys to GitHub Pages.

### One-time setup

1. Repository **Settings тЖТ Pages тЖТ Build and deployment тЖТ Source =
   GitHub Actions**.
2. Push to `main`. The workflow runs, builds, and publishes.
3. Site is live at `https://dhananjay-chaturvedi.github.io/dbassistant/`.

### Custom domain

1. **Settings тЖТ Pages тЖТ Custom domain** тЖТ enter your domain (e.g. `docs.example.com`).
2. Add a `CNAME` record at your DNS provider pointing to
   `dhananjay-chaturvedi.github.io`.
3. GitHub Pages provisions Let's Encrypt TLS automatically.

Update `astro.config.mjs` тАФ set `site` to your domain and remove `base`:

```js
site: 'https://docs.example.com',
// base: '/dbassistant/',  // remove when using a root custom domain
```

Re-run the workflow.

## Option 2 тАФ Cloudflare Pages

1. Sign in to [Cloudflare Pages](https://pages.cloudflare.com/).
2. **Create a project тЖТ Connect to Git тЖТ select this repo**.
3. Build configuration:
   - Framework preset: **Astro**
   - Build command: `npm run build`
   - Build output directory: `dist`
   - Root directory: `website`
   - Environment variable: `NODE_VERSION=20`
4. Save. First deploy starts immediately.
5. The site is live at `https://<project>.pages.dev`.

Attach a custom domain in **Custom domains тЖТ Set up a custom domain**.

## Option 3 тАФ Netlify

1. **Add new site тЖТ Import an existing project тЖТ pick this repo**.
2. Build settings:
   - Base directory: `website`
   - Build command: `npm run build`
   - Publish directory: `website/dist`
   - Node version: `20`
3. Deploy. Live at `https://<random>.netlify.app`.

## Option 4 тАФ Vercel

1. **Add New Project тЖТ import repo**.
2. Root directory: `website`.
3. Framework: **Astro** (auto-detected).
4. Deploy.

## Option 5 тАФ S3 + CloudFront

```bash
cd website
npm ci && npm run build
aws s3 sync dist/ s3://your-bucket --delete --acl public-read
aws cloudfront create-invalidation \
    --distribution-id E... --paths "/*"
```

Set the CloudFront default-root-object to `index.html` and use a
S3-Origin behaviour with default-TTL=300 for cache control.

## Updating the `site` URL

When you attach a domain, update `astro.config.mjs`:

```js
site: 'https://your-domain.example.com',
```

This affects:

- Canonical URLs in HTML `<head>`
- Open Graph absolute image paths
- `sitemap.xml` (generated by Astro)
- Internal links produced by Starlight

## CDN-friendly headers

Static hosts above all set immutable cache headers for assets and
short cache TTL for HTML by default. No extra configuration needed.

## Disabling Pagefind search (optional)

Search is included by default. To remove, edit `astro.config.mjs`:

```js
starlight({
  pagefind: false,
  ...
})
```
